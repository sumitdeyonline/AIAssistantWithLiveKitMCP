# AI Voice Assistant with LiveKit & MCP

This project implements a fully functional, low-latency Voice AI Assistant using the **LiveKit Agents SDK (v1.5+)**. It features advanced capabilities including Speech-to-Text (STT), Text-to-Speech (TTS), Large Language Model (LLM) orchestration, and dynamic API integrations using the **Model Context Protocol (MCP)**.

## Project Architecture

This repository contains multiple approaches to building and deploying a voice agent:

1. **`agent.py` (Basic Function Calling)**
   A standard LiveKit background worker. It utilizes the `@llm.function_tool` decorator directly inside the script to fetch real-time weather information from Open-Meteo.
   
2. **`weather_mcp.py` & `agent_mcp.py` (MCP Integration)**
   An advanced architecture demonstrating the **Model Context Protocol**. 
   - `weather_mcp.py` runs an isolated, standalone Python `FastMCP` server serving our weather tool.
   - `agent_mcp.py` connects to it dynamically over Standard I/O using `MCPServerStdio`. This allows your AI agent to securely discover and execute external tools without bloating the core agent codebase.

3. **`app.py` (Streamlit Web UI)**
   A lightweight Python frontend utilizing Streamlit that generates secure JWT access tokens for your LiveKit room. It embeds the official `livekit-client` vanilla JS SDK in the browser to seamlessly transport microphone audio to your backend AI agent.

## 🚀 Getting Started

### Prerequisites

Ensure you have [uv](https://github.com/astral-sh/uv) configured on your machine for fast Python environment management.

Install all dependencies:
```bash
uv pip install -r requirements.txt
```

### Authentication Keys
Create a `.env` file in the root directory equipped with your API keys:

```ini
LIVEKIT_URL="wss://your-project.livekit.cloud"
LIVEKIT_API_KEY="..."
LIVEKIT_API_SECRET="..."

OPENAI_API_KEY="..."
DEEPGRAM_API_KEY="..."
CARTESIA_API_KEY="..."
```

*(Note: LiveKit credentials can be generated by creating a free project on [LiveKit Cloud](https://cloud.livekit.io/)).*

## 🏃 Running the Application

This is a bipartite system consisting of a Backend worker and a Frontend UI.

### 1. Start the Voice Agent Backend
Pick **one** of the backend options below and run it in your terminal. It will stay running silently, waiting for users to connect.

**Basic Agent:**
```bash
uv run agent.py dev
```
**MCP Powered Agent:**
```bash
uv run agent_mcp.py dev
```

### 2. Connect via the Frontend
Once your background worker is active, you can talk to the agent in two ways:

**Option A: Streamlit UI (Local)**
Open a *new* terminal window and run:
```bash
uv run streamlit run app.py
```
This will automatically pop open a web browser. Click the **Connect & Speak** button, allow microphone permissions, and start talking!

**Option B: LiveKit Sandbox**
Navigate to the [LiveKit Agents Sandbox](https://agents-playground.livekit.io/), connect using your same LiveKit Cloud project tokens, and talk.
